# Track: Interview Craft (IC1/IC2/IC5 in weeks 1–3; drilled forever via /mock)

The articulation track — fixes "I know it but can't say it." Nothing here is knowledge; it's all reps.

| # | Module | Objectives |
|---|---|---|
| IC1 | Answer-first framing | The structure for EVERY technical answer: conclusion → 2–3 reasons → concrete example → stop talking. Kill filler, hedging ("I think maybe…"), and lede-burying. Drilled in every /teach close-out. |
| IC2 | STAR stories + senior signals | Mine 4–6 yrs of real work into 8–10 stories in [stories/story-bank.md](../../stories/story-bank.md), each mapped to a signal: ownership, conflict, ambiguity, mentoring, failure/learning, impact, AI adoption. Each story: 90-second spoken version, drilled aloud. |
| IC3 | Defining & evidencing "senior" | A crisp personal definition (scope of ownership, multiplying others, judgment under ambiguity, outcome accountability) + which bank stories prove each element. Kills the exact question that's been killing him. |
| IC4 | Asking questions | The design-interview question checklist (SD1) as a reflex, plus reverse-interview questions that signal seniority ("how do you measure success for this role?"). |
| IC5 | CV + narrative for the sponsorship market | Rewrite CV around the AI-product-engineer identity: outcome-led bullets (metric + scope), one coherent story arc, sponsorship-friendly signals. Target companies from the GOV.UK licensed sponsor register — never apply where sponsorship is impossible. |
| IC6 | Talking about AI work credibly | Vocabulary and honesty calibration: what he's built vs used vs read about. Overclaiming AI experience is detected instantly in 2026 interviews; the credible version of a small true thing beats an inflated vague thing. |

Grounding: [MISSION.md](../MISSION.md) · Session rules: [NOTES.md](../NOTES.md) · Status: [tracker/progress.md](../tracker/progress.md)

## Depth map — the antidote to "advanced everything"

| Tier | Bar | Topics |
|---|---|---|
| **1 — Advanced** (can whiteboard & teach it) | Survives 3 levels of interviewer follow-up | JS/TS internals (event loop, closures, async, type system†) · React rendering & performance (incl. signals vs VDOM as follow-up material) · Node fundamentals · Agentic AI · Product-scale system design |
| **2 — Working** (solve & discuss trade-offs) | Solves the problem, states the trade-offs unprompted | DSA + complexity to LC-medium · SQL (modeling, joins, indexing, transactions) · API design · HTML/CSS (semantics, layout, a11y) · Build tooling & bundlers · Service workers/worker threading · WebSocket protocol layer |
| **3 — Conversational** (when-to-use fluency) | Sounds informed for 3 minutes, knows the boundaries of own knowledge | NoSQL families · caching/queues/scaling vocabulary · testing strategy · security basics · Web components/shadow DOM · view transitions |

† **F4 test-out:** tiers are set by interview reality, but study hours can be reclaimed on evidence — TypeScript gets a week-1 interview-grade spoken assessment; pass = review-only, hours fund F11/F12.

## 24-week schedule (started 2026-07-07)

- **Weeks 1–2 — Setup + funnel triage:** baseline diagnostic · **F4 TypeScript test-out assessment** · story bank v1 (IC2) · CV rewrite around the identity (IC5) · funnel tracker live, targeting licensed sponsors
- **Weeks 1–6 — Foundations pass:** F1→F10 via `/teach`; week 6 = diagnostic mock battery (one of each round) sets Phase B priorities
- **Weeks 3–20 — Portfolio project (~4h/wk):** agentic app in TS — concept confirmed at week-3 kickoff; candidate: multi-agent job-search assistant (research, CV-tailoring, interview-prep agents; tool use, memory, eval harness)
- **Weeks 7–20 — Output-first cycles:** 2-week cycles pairing a track with its mock type — system design ×3, AI/agents ×2, coding ×1, behavioral ×1 — reordered by mock-log evidence, not by plan loyalty
- **Weeks 21–24 — Final sprint:** full-loop simulations · gap-driven review from trackers · spaced-repetition consolidation · project demo polish

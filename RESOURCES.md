# AI Product Engineer (Interview Prep) Resources

## Knowledge

- [javascript.info — The Modern JavaScript Tutorial](https://javascript.info)
  The single best structured JS deep-dive. Use for: event loop, closures, prototypes, async — the Tier-1 JS modules.
- [MDN Web Docs](https://developer.mozilla.org)
  Canonical reference for JS/HTML/CSS/browser APIs. Use for: verifying any web platform claim before teaching it.
- [TypeScript Handbook](https://www.typescriptlang.org/docs/handbook/intro.html)
  Official TS docs. Use for: generics, narrowing, utility types, structural typing (module F4).
- [react.dev — Learn React](https://react.dev/learn)
  Official docs incl. rendering model and escape hatches. Use for: reconciliation, hooks semantics, performance.
- [Node.js Learn guides](https://nodejs.org/en/learn)
  Official Node fundamentals incl. event loop specifics. Use for: the Node Tier-1 modules.
- [Anthropic — Building Effective Agents](https://www.anthropic.com/engineering/building-effective-agents)
  The reference essay on agent patterns (workflows vs agents, orchestration). Use for: modules A4–A5 and portfolio architecture decisions.
- [Claude Docs](https://docs.claude.com)
  API reference for the portfolio project: tool use, streaming, structured output, Agent SDK.
- [SQLBolt](https://sqlbolt.com)
  Interactive SQL lessons. Use for: F9 practice loops.
- [Use The Index, Luke](https://use-the-index-luke.com)
  Indexing and query performance explained properly. Use for: the indexing half of F9 — interviewers love this gap.
- [_Designing Data-Intensive Applications_ — Kleppmann](https://dataintensive.net)
  The system design bible. Use SELECTIVELY (chapters on replication, partitioning, transactions) — reading it cover-to-cover is off-mission at Tier 2/3.
- [System Design Primer (GitHub)](https://github.com/donnemartin/system-design-primer)
  Free structured system design vocabulary. Use for: SD2 core vocabulary.
- [NeetCode Roadmap](https://neetcode.io/roadmap)
  Curated DSA problem ordering. Use for: F7–F8 problem selection (solve in TypeScript).
- [GOV.UK — Register of Licensed Sponsors (Workers)](https://www.gov.uk/government/publications/register-of-licensed-sponsors-workers)
  The official list of UK companies that CAN sponsor. Use for: funnel targeting — apply where sponsorship is possible, not hopeful.

## Wisdom (Communities)

- [r/ExperiencedDevs](https://reddit.com/r/ExperiencedDevs)
  High-signal for senior-role framing and career mechanics. Use for: sanity-checking "what does senior mean" answers.
- [r/cscareerquestionsUK](https://reddit.com/r/cscareerquestionsuk)
  UK-specific market/visa intel. Use for: sponsorship-market reality checks, salary bands.
- London meetups (LNUG, AI engineering meetups via meetup.com)
  Use for: real humans to practice the identity pitch on, and warm referrals — the strongest funnel channel for sponsorship candidates.

## Gaps

- A vetted source for **frontend system design interviews** (SD4) — evaluate GreatFrontEnd or similar before week 7.
- A structured source for **evals** (A6) — Anthropic/OpenAI cookbook eval examples to be vetted when the project reaches that stage.
- Peer mock-interview partners (human stakes) — revisit around week 8.
